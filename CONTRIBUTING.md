## Submitting a Pull Request

1. Create a descriptive branch name:

    ```bash
    git checkout -b feature/new-feature
    # or
    git checkout -b fix/bug-name
    ```

2. Implement your changes following the project's coding style.

3. Commit your changes with a clear and meaningful message:

    ```bash
    git commit -m "feat: add new feature X"
    ```

4. Push your branch:

    ```bash
    git push origin feature/new-feature
    ```

5. Open a Pull Request and include:
   - A clear description of what was changed
   - The reason for the change
   - Any relevant context for reviewers
